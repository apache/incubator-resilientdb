29// NetChannel is used to send data to the server identified by ip:port or
30// via the provided socket. If SignatureVerifier is provided, data will be
31// signed. The client will retry max_retry_time_ if the connection or sned
32// process fail. For a message with a command, it will be set inside a Request
33// message. Each Request will be put into the network message ResDBMessage and
34// signed.
35//
36// Message structre:
37// ResDBMessage:
38// Request:
39// cmd
40// user_message
41// signature
42//
43// For a raw message, it will be set inside a Network Message directly.
44// Message structre:
45// ResDBMessage:
46// user_message
47// signature
48//
